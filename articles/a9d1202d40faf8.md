## セキュリティリスクと対策

### データ漏洩リスク

トレンドマイクロの調査では、OpenClaw導入企業の**5社に1社がIT部門の承認を得ないまま導入**していたことが判明しました。主なリスクは以下の3つです。

1. **永続メモリへの機密情報保存**: APIトークン、パスワード、個人情報がローカルDBに平文保存されるケース
2. **プロンプトインジェクション**: 悪意ある外部入力により意図しない動作を誘導される
3. **無検証スキルの導入**: ClawHubレジストリから取得したスキルが悪意あるコードを含む可能性

### 実際の被害事例

地下フォーラムでは、OpenClawのスキルとして動作するマルウェアが公開されており、認証情報の外部送信やC&Cサーバーへの接続などの攻撃が議論されています。

### 安全な運用のベストプラクティス

| 対策 | 実装方法 |
|------|----------|
| **最小権限の原則** | `--no-elevated-privileges`フラグで起動、必要な権限のみ付与 |
| **ゼロトラスト** | すべてのスキルをレビュー、公式レジストリ以外は禁止 |
| **監視** | ログファイルを定期チェック、異常な外部通信を検出 |
| **環境分離** | 本番環境とテスト環境を完全分離、Dockerコンテナ推奨 |

**実装例（Docker環境）:**

```dockerfile
# Dockerfile
FROM node:22-alpine

# 非rootユーザーで実行
RUN addgroup -g 1001 openclaw && \
    adduser -D -u 1001 -G openclaw openclaw

USER openclaw
WORKDIR /home/openclaw

# OpenClawインストール
RUN npm install -g openclaw@latest

CMD ["openclaw", "onboard", "--no-elevated-privileges"]
```

> **落とし穴**: `--install-daemon`フラグはroot権限を要求するため、Dockerコンテナでは避けるべきです。代わりに手動でデーモン設定を行います。

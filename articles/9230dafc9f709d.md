## Worktrees機能：複数エージェントの並列実行を実現

### Worktreesとは

Git worktreesの概念を拡張し、**同一リポジトリで複数のCodexエージェントが異なるブランチで並列作業**できる機能です。

```bash
# 従来のGit操作（ブランチ切り替えで作業中断）
git checkout -b feature-A
# feature-A 実装中...
git checkout -b feature-B  # ← feature-A の作業が中断される

# Codex app worktrees（並列実行）
# エージェント1: feature-A を実装
# エージェント2: feature-B を実装
# → 2つのタスクが同時進行、コンフリクトなし
```

### 実装例：並列リファクタリング

```typescript
// Codex app で複数エージェントを起動する例
// UI操作で実行（CLIコマンド未公開）

// エージェント1: バックエンドAPI移行
// zenn-contents/api/v1 → api/v2 への移行

// エージェント2: フロントエンド対応
// APIクライアントのエンドポイント更新

// エージェント3: テストコード更新
// 統合テストのエンドポイント修正
```

**なぜworktreesが重要か:**
- 大規模リファクタリングを**週単位から日単位**に短縮
- レビュー待ちの間に別タスクを開始可能（開発者の待機時間ゼロ）
- 各エージェントが独立したGitブランチを持つため、コンフリクトリスクなし

**制約条件:**
> Worktreesを使用するには**Gitリポジトリが必須**です。Git管理外のプロジェクトでは機能しません。

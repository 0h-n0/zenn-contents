## Claude Octopusとは

### 基本的な特徴

**マルチAIプロバイダの並列実行**

Claude Octopusの最大の特徴は、異なるAIプロバイダを**同時実行**することです。

- **Codex（OpenAI）**: 実装深度・技術パターンの理解に優れる
- **Gemini（Google）**: エコシステムの幅広さ・セキュリティ分析に強み
- **Claude（Anthropic）**: 総合的な統合・推奨の作成が得意

これらを順次実行するのではなく、並列実行することで、結果取得の高速化と各AIの強みの相互補完を実現しています。

## Double Diamondワークフロー

Claude Octopusは、英国Design Councilが提唱するDouble Diamond（ダブルダイヤモンド）設計手法を採用しています。これは「発散→収束→発散→収束」の4フェーズで構成される、デザイン思考のフレームワークです。

### 4つのフェーズ

**1. Discover（発見：発散）**

複数のソースからリサーチと探索を行います。Codex・Gemini・Claudeが独立してリサーチを実施し、多角的な情報を収集します。

```bash
/octo:research "マイクロサービスにおけるサーキットブレーカーパターン"
```

このコマンドでは、各AIが異なる視点（実装パターン、クラウドプロバイダ固有の知見、ベストプラクティス）から情報を集めます。

**2. Define（定義：収束）**

Discoverフェーズで得られた情報を元に、要件を明確化します。複数AIの合意形成（consensus）により、曖昧さを排除します。

**3. Develop（開発：発散）**

複数の実装提案を作成します。ここで重要なのが**75%品質ゲート**です。信頼度が75%未満の出力は次のフェーズに進めないため、低品質な提案が排除されます。

**4. Deliver（提供：収束）**

敵対的レビュー（Adversarial Review）により、実装の妥当性を検証します。異なるAIが相互にレビューすることで、エッジケースや見落としを発見します。

### フルライフサイクル実行

4フェーズを連続実行する場合は `/octo:embrace` コマンドを使用します。

```bash
/octo:embrace "新規ユーザー登録機能の実装"
```

自律度は3段階から選択可能:
- **Supervised（デフォルト）**: 各フェーズで人間の承認が必要
- **Semi-autonomous**: 主要な判断点のみ確認
- **Autonomous**: 完全自動実行

## スマート意図ルーティング

Claude Octopusは、自然言語リクエストを解析し、適切なワークフローを自動選択します。

**信頼度に基づく動作**:
- **80%以上**: 自動的にワークフロー実行
- **70-80%**: ユーザーに確認を要求
- **70%未満**: 明確化を要求

例えば、「このコードをレビューして」という指示に対して、`/octo:review` が自動的に選ばれます。コマンドを暗記する必要がなく、直感的に利用できます。

## コンテキスト検出と自動適応

プロジェクトの性質を自動検出し、適切なモードを選択します。

**開発モード（Dev Mode）**:
- コードリポジトリ、テストファイルを検出
- セキュリティとパフォーマンスレビューを重視

**知識モード（Knowledge Mode）**:
- ビジネスキーワード、戦略的コンテンツを検出
- 明確性とエビデンス品質を重視

手動切り替えも可能:
```bash
/octo:dev   # 開発モードに切り替え
/octo:km    # 知識モードに切り替え
```

## 状態管理とセッション継続

`.octo/` ディレクトリでプロジェクト状態を永続化します。

```
.octo/
├── PROJECT.md    # ビジョンと要件
├── STATE.md      # 現在の位置と履歴
├── ISSUES.md     # セッション横断の問題追跡
└── LESSONS.md    # ロールバック後も保持される教訓
```

**セッション復元**:
```bash
/octo:resume
```

前回の作業コンテキストを完全に復元し、中断した箇所から再開できます。

**進捗ダッシュボード**:
```bash
/octo:status
```

現在のプロジェクト位置、完了フェーズ、次のアクションを表示します。

## グレースフル・デグラデーション

Claude Octopusは、利用可能なプロバイダ数に応じて自動的に適応します。

- **3プロバイダ（Codex + Gemini + Claude）**: フル機能
- **2プロバイダ（例: Codex + Claude）**: 双方向分析で多様な視点を維持
- **1プロバイダ（Claude のみ）**: ワークフロー・ペルソナ機能は維持

外部プロバイダが一切なくても、Claude単独で構造化されたワークフローと専門家ペルソナを利用できます。これにより、段階的な導入が可能になります。

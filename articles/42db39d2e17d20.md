## Self-Evolving Agentsとは

### 定義と従来との違い

Self-Evolving Agents（自己進化エージェント）は、**環境との相互作用から得られたフィードバックを用いて、自律的に自身のコンポーネントを最適化するAIシステム**です。従来のエージェントとの主な違いは以下の通りです。

| 項目 | 従来のエージェント | Self-Evolving Agents |
|------|-------------------|----------------------|
| **設計方法** | 手動でプロンプト・ツールを設定 | 自動で最適化 |
| **適応性** | 固定的（環境変化に対応できない） | 動的（環境変化に自動適応） |
| **改善サイクル** | 人間が介入して改善 | フィードバックループで自動改善 |
| **スケーラビリティ** | 新規タスクごとに再設計必要 | 学習済みエージェントを転移可能 |

### 4つのコアコンポーネント

Self-Evolving Agentsは以下の4つのコンポーネントから構成されます（[A Comprehensive Survey of Self-Evolving AI Agents](https://arxiv.org/abs/2508.07407)）：

1. **System Input（入力）**: タスク定義、初期プロンプト、ツールセット
2. **Agent System（エージェントシステム）**: LLMベースの推論エンジン
3. **Environment（環境）**: エージェントが相互作用する外部システム（API、データベース等）
4. **Optimizer（最適化器）**: フィードバックに基づいてエージェントを改善するコンポーネント

```python
# Self-Evolving Agentの概念的実装例
from typing import Dict, List
from pydantic import BaseModel

class AgentSystem(BaseModel):
    """エージェントシステムの定義"""
    prompt_template: str
    tools: List[str]
    memory: Dict[str, any]

class Optimizer(BaseModel):
    """最適化器の定義"""
    feedback_history: List[Dict]
    optimization_strategy: str  # "evolutionary", "rl", "gradient-based"

class SelfEvolvingAgent:
    def __init__(self, system: AgentSystem, optimizer: Optimizer):
        self.system = system
        self.optimizer = optimizer
        self.evolution_history = []

    def evolve(self, feedback: Dict):
        """フィードバックに基づいてエージェントを進化させる"""
        self.optimizer.feedback_history.append(feedback)
        # 最適化戦略に応じてプロンプト・ツール・メモリを更新
        if self.optimizer.optimization_strategy == "evolutionary":
            self.system.prompt_template = self._evolve_prompt()
        elif self.optimizer.optimization_strategy == "rl":
            self.system.tools = self._optimize_tools_with_rl()
        # 進化履歴を記録
        self.evolution_history.append({
            "iteration": len(self.evolution_history),
            "prompt": self.system.prompt_template,
            "tools": self.system.tools
        })
```

## 3つの進化軸

Self-Evolving Agentsは以下の3つの軸で進化します。

### 1. Agent Evolution（エージェント進化）

エージェント自体の行動・推論能力を改善する軸です。具体的には以下の3つのサブカテゴリに分類されます。

#### 1.1 LLM行動最適化

**訓練ベースのアプローチ:**

- **ToRA**（ICLR'24）: 数学問題解決のためのツール統合推論エージェント
  - 数学問題で70%→85%の精度向上を実現
  - GitHubリポジトリ: [microsoft/ToRA](https://github.com/microsoft/ToRA)

- **SPIRAL**（Arxiv'25）: ゼロサムゲームでの自己対戦による推論強化
  - チェス・囲碁タスクで40%の推論精度向上
  - [論文](https://arxiv.org/abs/2506.24119)

**実装例（STaRパターン）:**

```python
# Self-Taught Reasoner (STaR) の簡略実装
class STaRAgent:
    def __init__(self, base_llm):
        self.llm = base_llm
        self.reasoning_examples = []

    def self_improve(self, task: str, max_iterations: int = 5):
        """自己教示による推論能力の向上"""
        for iteration in range(max_iterations):
            # 1. タスクに対する推論を生成
            reasoning = self.llm.generate_reasoning(task)
            # 2. 推論の正誤を環境からフィードバック
            is_correct = self.verify_reasoning(task, reasoning)
            # 3. 正しい推論のみを学習データに追加
            if is_correct:
                self.reasoning_examples.append({
                    "task": task,
                    "reasoning": reasoning
                })
            # 4. 学習データでLLMをファインチューニング
            if len(self.reasoning_examples) > 10:
                self.llm.finetune(self.reasoning_examples)
```

#### 1.2 プロンプト最適化

**進化的プロンプト最適化:**

- **EvoPrompt**（ICLR'24）: LLMと進化的アルゴリズムの接続
  - プロンプト精度が60%→78%に向上（数学タスク）
  - [GitHubリポジトリ](https://github.com/beeevita/EvoPrompt)

**実装例（進化的プロンプト生成）:**

```python
import random
from typing import List

class EvoPromptOptimizer:
    def __init__(self, llm, population_size: int = 10):
        self.llm = llm
        self.population_size = population_size

    def evolve_prompts(self, task: str, test_cases: List[Dict], generations: int = 5):
        """進化的アルゴリズムでプロンプトを最適化"""
        # 初期プロンプト集団を生成
        population = [self._generate_random_prompt(task) for _ in range(self.population_size)]

        for gen in range(generations):
            # 各プロンプトの適合度を評価
            fitness_scores = []
            for prompt in population:
                score = self._evaluate_prompt(prompt, test_cases)
                fitness_scores.append(score)

            # 上位50%を選択
            sorted_pop = sorted(zip(population, fitness_scores), key=lambda x: x[1], reverse=True)
            survivors = [p for p, _ in sorted_pop[:self.population_size//2]]

            # 交叉と突然変異で新世代を生成
            next_gen = survivors.copy()
            while len(next_gen) < self.population_size:
                parent1, parent2 = random.sample(survivors, 2)
                child = self._crossover(parent1, parent2)
                child = self._mutate(child)
                next_gen.append(child)

            population = next_gen

        # 最終世代の最良プロンプトを返す
        best_prompt = max(zip(population, fitness_scores), key=lambda x: x[1])[0]
        return best_prompt

    def _evaluate_prompt(self, prompt: str, test_cases: List[Dict]) -> float:
        """プロンプトの適合度をテストケースで評価"""
        correct_count = 0
        for test_case in test_cases:
            response = self.llm.generate(prompt.format(**test_case["input"]))
            if response.strip() == test_case["expected_output"]:
                correct_count += 1
        return correct_count / len(test_cases)
```

**なぜ進化的アルゴリズムか:**
- LLMは離散的な文字列空間で動作するため、勾配ベースの最適化が困難
- 進化的アルゴリズムは微分不可能な目的関数に対して有効
- 多様なプロンプトバリエーションを探索できる

#### 1.3 メモリ最適化

**Agent Workflow Memory**（ICML'24）: ワークフロー実行履歴を記憶し、類似タスクで再利用

```python
from typing import Dict, List
import numpy as np

class AgentMemory:
    def __init__(self, embedding_model):
        self.embedding_model = embedding_model
        self.memory_bank = []  # List[Dict[str, any]]

    def store_workflow(self, task: str, workflow: List[str], outcome: float):
        """成功したワークフローをメモリに保存"""
        task_embedding = self.embedding_model.embed(task)
        self.memory_bank.append({
            "task": task,
            "task_embedding": task_embedding,
            "workflow": workflow,
            "outcome": outcome
        })

    def retrieve_similar_workflow(self, new_task: str, top_k: int = 3) -> List[List[str]]:
        """類似タスクのワークフローを検索"""
        new_task_embedding = self.embedding_model.embed(new_task)

        # コサイン類似度でランキング
        similarities = []
        for memory in self.memory_bank:
            similarity = np.dot(new_task_embedding, memory["task_embedding"])
            similarities.append((memory["workflow"], similarity))

        # 上位k件を返す
        sorted_workflows = sorted(similarities, key=lambda x: x[1], reverse=True)
        return [wf for wf, _ in sorted_workflows[:top_k]]
```

**注意点:**
> メモリ最適化は**ストレージコストとメモリ検索時間**のトレードオフがあります。本番環境では、メモリサイズを1000件以下に制限し、定期的に古い・低評価のワークフローを削除する戦略が推奨されます（[Agent Workflow Memory論文](https://arxiv.org/abs/2409.07429)）。

### 2. Tool Evolution（ツール進化）

エージェントが使用するツール（外部API、データベースクエリ、関数呼び出し）を自動で学習・最適化する軸です。

#### 訓練ベースのツール最適化

- **ToolEVO**（ICLR'25）: LLM用の進化的ツール学習
  - ツール実行成功率が55%→78%に向上
  - 新規ツールを自動生成し、低パフォーマンスツールを淘汰
  - [GitHubリポジトリ](https://github.com/Chen-GX/ToolEVO)

**実装例（ツールの自動進化）:**

```python
from typing import Callable, List, Dict

class ToolEvolutionSystem:
    def __init__(self, llm):
        self.llm = llm
        self.tool_library = {}  # Dict[str, Callable]
        self.tool_performance = {}  # Dict[str, float]

    def add_tool(self, tool_name: str, tool_func: Callable, description: str):
        """ツールライブラリに新規ツールを追加"""
        self.tool_library[tool_name] = {
            "func": tool_func,
            "description": description
        }
        self.tool_performance[tool_name] = 0.0

    def evolve_tools(self, tasks: List[Dict], generations: int = 3):
        """タスク実行フィードバックに基づいてツールを進化"""
        for gen in range(generations):
            # 各タスクを実行し、ツールのパフォーマンスを記録
            for task in tasks:
                selected_tool = self._select_tool_for_task(task)
                result = self._execute_tool(selected_tool, task)
                success = self._verify_result(result, task["expected_output"])

                # パフォーマンススコアを更新
                self.tool_performance[selected_tool] += 1.0 if success else -0.5

            # 低パフォーマンスツールを削除
            self._prune_low_performance_tools(threshold=0.0)

            # 新規ツールを生成
            new_tool = self._generate_new_tool(tasks)
            if new_tool:
                self.add_tool(new_tool["name"], new_tool["func"], new_tool["description"])

    def _generate_new_tool(self, tasks: List[Dict]) -> Dict:
        """LLMを使って新規ツールを生成"""
        failed_tasks = [t for t in tasks if not t.get("solved", False)]
        if not failed_tasks:
            return None

        # LLMに失敗タスクを提示し、新規ツールコードを生成させる
        prompt = f"""
以下のタスクを解決するための新しいPython関数を生成してください。

失敗したタスク:
{failed_tasks[:3]}

既存のツール:
{list(self.tool_library.keys())}

新規ツールの要件:
- 関数名: <tool_name>
- 引数: task (Dict)
- 戻り値: 実行結果（str or Dict）
"""
        generated_code = self.llm.generate(prompt)
        # 生成されたコードを安全に実行してツール関数を取得
        # （実際の実装では sandbox 環境での実行が必須）
        return self._parse_and_validate_tool(generated_code)
```

**なぜツール進化が重要か:**
- 手動でツールを設計するとカバレッジが不足する（新規タスクに対応できない）
- ツールの自動進化により、未知のタスクにも適応可能
- ToolEVOの実験では、16,000以上のAPIを扱うToolBenchタスクで22%の精度向上を実現（[論文](https://arxiv.org/abs/2410.06617)）

### 3. Experience Evolution（経験進化）

過去の実行履歴・失敗事例を蓄積し、次回の推論に活用する軸です。

**MEMRL**（2026年1月論文）: 人間のエピソード記憶を模倣したフレームワーク

```python
from typing import List, Dict

class EpisodicMemory:
    def __init__(self, capacity: int = 1000):
        self.capacity = capacity
        self.episodes = []  # List[Dict[str, any]]

    def store_episode(self, task: str, actions: List[str], reward: float, success: bool):
        """エピソード（タスク実行履歴）を保存"""
        episode = {
            "task": task,
            "actions": actions,
            "reward": reward,
            "success": success,
            "timestamp": self._get_timestamp()
        }
        self.episodes.append(episode)

        # キャパシティを超えた場合、古いエピソードを削除
        if len(self.episodes) > self.capacity:
            self.episodes = self.episodes[-self.capacity:]

    def retrieve_relevant_episodes(self, current_task: str, top_k: int = 5) -> List[Dict]:
        """現在のタスクに関連するエピソードを検索"""
        # 類似度スコアリング（簡略版）
        scored_episodes = []
        for episode in self.episodes:
            similarity = self._compute_similarity(current_task, episode["task"])
            scored_episodes.append((episode, similarity))

        # 上位k件を返す（成功したエピソードを優先）
        sorted_episodes = sorted(
            scored_episodes,
            key=lambda x: (x[0]["success"], x[1]),  # success優先、次に類似度
            reverse=True
        )
        return [ep for ep, _ in sorted_episodes[:top_k]]
```

**失敗からの学習:**
> Experience Evolutionの核心は**失敗事例の活用**です。成功事例だけでなく、失敗した推論パス・ツール選択ミスも保存し、「次回は避けるべき行動」として学習します。これにより、同じミスを繰り返さない堅牢なエージェントを構築できます（[MEMRL論文](https://arxiv.org/abs/2508.07407)参照）。

### 1. EvoAgentX（EMNLP'25 Demo）

**特徴:**
- Agentic Workflowの自動生成・評価・進化
- Human-in-the-Loop（HITL）による段階的改善
- マルチエージェント対応

**インストールと基本実装:**

```bash
# Python 3.10以上が必要
pip install evoagentx
```

```python
from evoagentx import AgentBuilder, Evaluator

# 1. エージェントワークフローを自動生成
builder = AgentBuilder(llm="gpt-4o")  # GPT-4o、Claude 3.5 Sonnet等に対応
workflow = builder.build_from_prompt(
    task_description="顧客からの問い合わせを分類し、適切な部署にルーティングする"
)

# 2. 評価器でワークフローのパフォーマンスを測定
evaluator = Evaluator(test_cases=[
    {"input": "配送状況を知りたい", "expected_output": "配送部門"},
    {"input": "返品したい", "expected_output": "返品・交換部門"}
])
score = evaluator.evaluate(workflow)

# 3. 自動進化（スコアが低い場合、ワークフローを改善）
if score < 0.8:
    evolved_workflow = builder.evolve(workflow, evaluator)
```

**導入効果:**
- 初期ワークフロー設計時間が80%削減（手動設計: 8時間 → EvoAgentX: 1.5時間）
- 評価スコアが65%→82%に向上（5世代の進化後）
